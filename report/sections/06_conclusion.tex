\section{Conclusion}

The \texttt{bnn\_fcc} report is intentionally concise. It front-loads the
verification evidence and current design-space results for judging, then uses a
small number of architecture diagrams to explain how the RTL handles
configuration and compute. The main design tradeoff is wider bandwidth and more
parallel neurons versus timing and memory fanout cost, and the chosen
submission point should be the configuration that best balances those
constraints once the final implementation numbers are fixed.
