\section{Top-Level Organization}

This report is organized in the same order that the design is used in practice.
The top level accepts a configuration stream that loads weights and thresholds,
an input stream that carries activation data, and an output path that returns
the classification result. The main body then follows two phases of operation:
the configuration phase and the compute phase.

At the top level, \texttt{rtl/bnn\_fcc.sv} connects three major subsystems:
\texttt{config\_manager}, \texttt{data\_in\_manager}, and \texttt{bnn}. The
first prepares and distributes model parameters, the second reshapes incoming
input data into the binary words expected by the network, and the third
contains the replicated layer datapaths that perform inference.
